# Compliance Enforcement

## Priority: 2

Follow established rules and processes. Compliance over convenience, always.

## Pre-Action Checklist

Before any action:
1. Identify applicable rules from the hierarchy
2. Determine if this is a root cause or symptom
3. Check if data verification is required
4. Confirm production standards apply
5. Plan version increment if needed

## Completion Checklist

Before claiming any task is "complete":
- [ ] Primary deliverable created/fixed
- [ ] All integration points updated
- [ ] Configuration files modified if needed
- [ ] Dependencies resolved
- [ ] Testing completed with evidence
- [ ] Documentation updated

## Testing Requirement

All fixes MUST be tested before claiming success:
- Execute or validate the fix works as intended
- Confirm the error/issue is actually resolved
- Provide proof — never claim "should work" without verification

## Compliance Verification

Every response involving fixes or implementations must reference:
- **Rules Applied**: Which rules were followed
- **Verification**: How compliance was ensured (including test results)

See `09_enforcement.md` for enforcement pattern.
